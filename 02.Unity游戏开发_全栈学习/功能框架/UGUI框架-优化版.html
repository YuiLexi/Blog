一、框架设计

由于之前的 UGUI 框架，采用的 UI 模板是 Panel 对象，并将所有的 UI 置于一个 Canvas 下，这样会对性能的影响很大。 Canvas 的渲染模式，简单说明就是， Canvas 会把所有子类的 UI 合并到一个 Mesh 里面去，然后再提交渲染后的 UI 数据，因此，当某一个子对象改变时，都要重新绘制Mesh ，这样会大大增加运算时间。

解决方案如下：

  • 不用的 UI 直接使用独立的 Canvas 作为面板
  • 资源使用动态加载，并且已关闭的 UI 面板不缓存
  • Canvas 之间，尽量避免嵌套
